# Contributing to AION v1

We appreciate contributions to improve AION! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch (`git checkout -b feature/YourFeature`)
4. Make your changes
5. Commit your changes (`git commit -m 'feat: Add YourFeature'`)
6. Push to the branch (`git push origin feature/YourFeature`)
7. Open a Pull Request

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting a PR, please test:
- Backend routes with appropriate data
- Frontend components in different browsers
- Error handling for edge cases

## Reporting Issues

- Check if the issue already exists
- Provide a clear description
- Include steps to reproduce
- Share error messages and screenshots if applicable

## Feature Requests

- Check the roadmap first
- Explain the use case
- Describe expected behavior
- Suggest implementation approach

## Questions?

Open a discussion or check existing issues for answers.

Thank you for contributing to AION!
